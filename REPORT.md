# Lab 6 实验报告草稿

## 一、实验名称

Lab 6：单周期 CPU 学号排序模拟实验

## 二、实验目的

1. 理解单周期 RISC-V CPU 的基本执行流程，包括取指、译码、执行、访存和写回。
2. 理解排序程序对控制器、ALU、立即数扩展、跳转与分支逻辑的功能需求。
3. 在仿真环境中运行学号排序程序，验证 CPU 能正确执行所需指令。
4. 为后续在 Nexys4 DDR 开发板上的单周期 CPU 和流水线 CPU 上板实验做准备。

## 三、实验内容

本实验提供了一个可直接在 `iverilog` 中运行的单周期 CPU 参考工程。CPU 读取指令存储器中的 RISC-V 排序程序，将 8 位学号按十进制数字从小到大排序，并把结果写回数据存储器指定地址。

本实验中使用的学号采用 BCD 编码形式保存，本次实验对应的学号为：

```text
02181164
```

对应的 32 位数据为：

```text
0x02181164
```

程序执行完成后，数据存储器中应得到：

```text
mem[0x180] = 0x02181164
mem[0x184] = 0x01112468
```

其中 `0x01112468` 表示将数字 0、2、1、8、1、1、6、4 重新按从小到大排列为 0、1、1、1、2、4、6、8 后的结果。

## 四、工程组成

`lab-6/source-sc` 目录下主要文件如下：

- `SCCPU.v`：单周期 CPU 顶层数据通路。
- `sccomp.v`：将 CPU、指令存储器和数据存储器连接起来的仿真顶层。
- `sccomp_tb.v`：测试平台，负责加载机器码、初始化内存并检查结果。
- `ctrl.v`：控制器，根据指令类型产生控制信号。
- `alu.v`：算术逻辑单元，完成加减、逻辑、比较和移位等运算。
- `EXT.v`：立即数扩展模块。
- `NPC.v`：下一条 PC 计算模块。
- `PC.v`：程序计数器。
- `RF.v`：寄存器堆。
- `dm.v`：数据存储器。
- `im.v`：指令存储器。
- `rv32_sid_sort_sim.asm`：学号排序汇编程序。
- `rv32_sid_sort_sim.dat`：对应的机器码文件。

## 五、实验原理分析

### 1. 排序算法原理

汇编程序本质上实现的是选择排序。程序每一轮从当前未处理的数字中找出最大值，并将该最大值交换到当前处理位置。由于数据以 BCD 方式压缩在一个 32 位寄存器中，每个十进制数字占 4 bit，因此程序需要不断使用掩码和移位操作，依次提取各个数字进行比较与交换。

排序过程中的核心寄存器是 `x15`，它保存当前正在被排序的 32 位学号数据。程序先把原始学号写入 `mem[0x180]`，排序结束后再把结果写入 `mem[0x184]`。

### 2. 寄存器分工

排序程序中各寄存器的主要用途如下：

- `x1`：保存 `swap` 子程序返回地址，后续还保存显示外设地址。
- `x2`：外层循环变量 `i`，排序结束后复用为显示地址寄存器。
- `x3`：内层循环变量 `j`。
- `x4`：外层掩码 `mask0`。
- `x5`：内层掩码 `mask1`。
- `x6`：交换时使用的复合掩码 `mask2`。
- `x7`：当前外层位置数字 `a`。
- `x8`：当前内层扫描到的数字 `b`。
- `x9`：`4 * i`。
- `x10`：`4 * j` 或 `4 * bestj`。
- `x11`：常量 `8`，表示学号数字个数。
- `x12`：当前轮最大值所在位置 `bestj`。
- `x13`：当前轮最大数字 `tmpMax`。
- `x14`：比较结果。
- `x15`：正在排序的学号数据 `sortedsid`。

### 3. 关键指令分析

本实验排序程序依赖的关键指令包括：

- 算术与逻辑指令：`add`、`addi`、`and`、`andi`、`or`、`ori`、`xori`
- 比较指令：`slt`
- 移位指令：`sll`、`slli`、`srl`
- 访存指令：`lw`、`sw`
- 分支指令：`beq`、`bne`
- 跳转指令：`jal`、`jalr`
- 高位立即数指令：`lui`

这些指令分别由以下模块支持：

- `ctrl.v` 负责识别指令类型并输出 `RegWrite`、`MemWrite`、`ALUSrc`、`WDSel`、`NPCOp`、`ALUOp` 和 `EXTOp` 等控制信号。
- `alu.v` 负责完成加法、减法、比较、逻辑运算和移位操作。
- `EXT.v` 负责 I/S/B/U/J 型立即数扩展。
- `NPC.v` 负责实现顺序执行、分支跳转、`jal` 和 `jalr` 的下一条地址计算。

### 4. `jal` 与 `jalr` 的作用

程序中 `jal` 和 `jalr` 的配合实现了子程序调用与返回。

调用 `swap` 子程序时，程序执行：

```asm
jal x1, swap
```

它的含义是：

1. 把当前指令的下一条地址 `PC + 4` 写入寄存器 `x1`。
2. 把程序跳转到 `swap` 标签对应的地址继续执行。

在 `swap` 子程序执行完成后，程序执行：

```asm
jalr x0, x1, 0
```

它的含义是：

1. 读取寄存器 `x1` 中保存的返回地址。
2. 计算跳转目标地址 `x1 + 0`。
3. 清除最低位后装入 PC，实现返回到主程序。

由于目的寄存器写成 `x0`，因此这条 `jalr` 不再保存新的返回地址，只负责跳回调用点。这样就完成了类似高级语言中“调用函数并返回”的过程。

### 5. 单周期 CPU 数据通路说明

单周期 CPU 的执行流程可以概括为：

1. `PC.v` 给出当前 PC。
2. `im.v` 根据 PC 取出指令。
3. `ctrl.v` 解析操作码和功能码，输出控制信号。
4. `RF.v` 读出源寄存器数据。
5. `EXT.v` 生成扩展后的立即数。
6. `alu.v` 完成运算、地址计算或条件比较。
7. `dm.v` 在 `lw/sw` 时完成数据读写。
8. `SCCPU.v` 根据 `WDSel` 决定写回 ALU 结果、内存数据或 `PC + 4`。
9. `NPC.v` 计算下一条 PC，实现顺序执行或跳转分支。

其中写回选择关系为：

- 普通算术/逻辑/移位/比较类指令写回 ALU 结果。
- `lw` 指令写回数据存储器读出的值。
- `jal` 和 `jalr` 写回 `PC + 4`，用于保存返回地址。

## 七、实验结论

本次实验成功完成了单周期 CPU 上的学号排序仿真。通过阅读与运行工程，可以确认该 CPU 已经支持排序程序所需的算术、逻辑、移位、比较、访存以及跳转分支指令，并能够正确执行包含子程序调用的完整 RISC-V 程序。

通过本实验，我进一步理解了以下内容：

1. 单周期 CPU 控制信号与数据通路之间的配合关系。
2. BCD 编码数据在寄存器中的组织方式，以及如何用掩码和移位提取数字。
3. 选择排序在汇编层面的实现方法。
4. `jal`/`jalr` 在子程序调用和返回中的作用。

本实验为后续将程序迁移到 FPGA 开发板上的单周期 CPU 和流水线 CPU 奠定了基础。
